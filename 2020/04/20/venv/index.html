Some thinking about venv

  • ‘venv’ is used to create a virtual environment, where we can use python seperate from the origianl env.

    • bin/activate is a file used for activating the virtual environment

  • Usage

    • make a working director (eg. myvenv)

      • mkdir -p(will check along the path, if not exist, it will create) venvs(means all my venvs)/myvenv
      • cd ./venvs/myvenv (go into the virtual env)

    • create venv

      • python3 -m venv .
        • . means the place you wang to create bin/ scripts/ ...

    • edit your codes, or git clone codes src

    • activate venv

      • source bin/activate

jupyter

  • jupyter has some user-defined properties, if we simply use ‘jupyter notebook’, it will use the default settings

    • eg. I used to use jupyter notebook --no-browser --port=xxxx on school server

  • Steps:

    • generate config files(jupyter_notebook_config.py), so that the original one will be overwritten

      • cmd: jupyter notebook --generate-config
      • Overwrite /root/.jupyter/jupyter_notebook_config.py with default config? [y/N]y
      • Writing default config to: /root/.jupyter/jupyter_notebook_config.py

    • edit the config file

      • vim /root/.jupyter/jupyter_notebook_config.py

  • jupyter notebook –generate-config –allow-root

Hexo

  • IF you happen to meet ‘fatal: Could not read from remote repository.’, then probably you have accidentally removed your rsa_key files

  • Solution

    • set name and email

      • git config –global user.name “yourname”
      • git config –global user.email “your@email.com“

    • comment the line of the target host ip (or remove .ssh/known_hosts)

    • ssh-keygen -t rsa -C ‘your@email.com(填写github对应的邮箱)’

      • Enter several times

    • Go to github

      • create new SSH KEY, and paste the connent in id_rsa.pub into it
